This is my best take on VS Code Shortcuts that I find useful for regular use.

1. Ctrl + / => For making comment of multiple lines of code, reducing repetitive single line comments.

2. Ctrl + D => Select multiple occurrence of a word. can work well with Ctrl + Shift + L to grab all matches of search at once, great for renaming multiple variables/strings at once.

3. Ctrl + Shift + L => Select all occurrences of the current selection Works hand‑in‑hand with Ctrl + D, one is incremental, the other is global.

4. Ctrl + Alt + Down / Up  => Adds a cursor above and below the origin cursor. Works well with Alt + Down / Up, to edit multiple lines simultaneously.

5. Alt + Up / Down => Move original  cursor/ code line above or below.

6. Ctrl + Shift + K => Delete current line. Works well with Ctrl + Alt + Down/Up to delete multiple lines at the same time.

7. Ctrl + P => Quick open file by name. Can be paired with Ctrl + Shift + O to immediately jump to function inside that file after opening.

8. Ctrl + Shift + O => To jump straight to a function/variable in the current file.

9. Ctrl + ' => Toggle Integrated Terminal. Can run tests/commands after editing code with multiple cursors.

10. Ctrl + Shift + \\ => Jump between matching brackets, especially when working with nested code.

Which shortcuts do you think will be the most useful in your daily work?
  => Most likely, Ctrl + D synergized with Ctrl + Shift + L, since it can edit multiple variables/strings especially longer code.

How can keyboard shortcuts improve your productivity?
  => Most likely Ctrl + /, as it trivialize making comments to either to make actual comments or to isolate certain blocks of code to exclude during testing.

These are my take on the 10 shortcuts that I find useful.
